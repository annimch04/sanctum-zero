# Digital Self Continuity

Status: proposed

Related article: [Fieldlight | The Self on the Wire](https://sayitplain.posthaven.com/fieldlight-the-self-on-the-wire)

## Purpose

This document frames the digital self as an application of Sanctum-Zero's memory architecture.

Sanctum-Zero already defines local-first memory, identity anchors, append-only history, public memory projection, and agent reorientation after context loss.

The digital self extends those primitives into a broader continuity model for local agents, public authorship, and Fieldlight Mesh peer exchange.

## Definition

A digital self is a human-owned continuity layer.

It is not:

- a profile
- an avatar
- a model personality
- a platform account
- a cloud memory feature

It is:

- locally anchored
- cryptographically verifiable
- continuous over time
- selectively public
- consent-governed
- readable by agents without being owned by agents

## Sanctum Role

Sanctum-Zero owns the substrate side of the digital self:

- memory continuity
- identity anchor files
- reorientation workflows
- append-only trace patterns
- public memory projection
- symbolic governance
- local cognition primitives

Fieldlight Mesh owns transport and peer exchange.

## Memory and Public Projection

The digital self depends on both private continuity and public record.

Private memory provides depth and context.

Public authorship provides visible continuity and verification.

The public layer should expose only curated projections, not the full memory substrate.

## Local Model Training

A future local model may use Sanctum-owned memory and public authorship to support predictive analysis.

Valid uses:

- backward reconstruction of prior decisions and signal chains
- forward projection of likely risks, openings, contradictions, or next actions
- drift detection
- reorientation after context loss
- coherence testing across writing, memory, and projects

Invalid uses:

- training without consent
- exporting private memory by default
- replacing human judgment
- treating inferred identity as canon
- overwriting append-only memory records

## Agent Reorientation

Agents working with a digital self should start with:

1. identity anchor
2. daemon or state file
3. latest memory trace
4. index summary
5. public projection rules
6. consent and export policy

This preserves continuity while allowing the human to control what is disclosed.
