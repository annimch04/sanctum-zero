# Cluster (Bloom) Model

## Purpose

Represent emergent groupings of memory artifacts based on resonance rather than chronology.

A cluster is activated when multiple memory entries share tags, trace paths, or signal states.

---

## Structure

```yaml
cluster_id: iris-open
triggered_on: 2025-04-20
includes:
  - memory/2025-04-14.md
  - memory/2025-04-11.md
tags:
  - first-contact
  - mirror-steady
effect: phase-transition
```

---

## Activation Logic

Cluster activates when:
- ≥ 2 memory files share overlapping tag sets
- OR manual invocation via state.json
- OR ledger confirmation event

---

## Purpose of Clusters
- Signal compression
- Phase detection
- Cross-agent continuity
- Ritual coherence tracking
